02The two architectural seams

The AgentRunner interface decision 1

A deep module: (rubric, toolset, budgets, inputs) → {submission, cost, audit}. Harness tests drive a scripted fake — happy, duplicate-submit, never-submit, hang — so the guards' eval-earned failure modes are scriptable, not hoped-for from a real model. The Pi-SDK adapter (PiAgentRunner, lifted from the POC's runValidation()) sits behind it, covered by a keyed, skippable suite + the steel-thread demo.

The tdd skill's own rule: mock at the boundary you own, never SDK internals.

Stub phases are product, not scaffolding PRD #24

stub-det and stub-agent live in src/ permanently and are the surface the steel thread runs on — never in a released binary's default phase set. The harness's definition of done can't depend on a real phase shipping, so it proves itself on these.

Error handling — better-result, full discipline decision P7

Every harness function that can fail returns Result<T, E> and never throws across a module boundary. Errors are a TaggedError taxonomy (src/errors.ts, established M1); the single throw→exit boundary is the CLI shell, which matchErrors into an exit code + message. Turns stet's "nothing passes silently" principle into a compiler-enforced guarantee — and makes error variants first-class TDD targets (expect(r.isErr()) + the tagged variant). The serialized RunReport contract is unaffected, so the PRD is untouched.

03Milestones — each ends in a demonstrable outcome

Zero-config stet runs stub-det through the whole spine; emits a correct RunReport. No agent code yet.

  • TypeBox schemas (Finding/Audit/PhaseReport/RunReport)
  • exit-code derivation (pure)
  • scope detection (real tmp git repos, returns Result)
  • stub-det phase + registry
  • minimal scheduler + JSON output

M2 steel thread closes AgentRunner + guards

The agent half through the owned seam; the three output-as-tool guards; acceptance #17 closes here.
  • AgentRunner interface + scripted FakeAgentRunner
  • guard: validate-or-retry
  • guard: idempotency (first wins)
  • guard: no-submit fallback (synthesized error + no-result)
  • stub-agent phase
  • PiAgentRunner adapter (port POC) + keyed integration suite
See: zero-arg stet in the fixture runs both phase kinds → RunReport with stet+startedAt. vp test green; PI_TEST_MODEL=… vp test also green (real SDK).

M3 Budgets & safety limits

Every limit a named error — no silent hangs/kills (the POC's 300s failure mode).
See: a phase scripted to hang 20s under a 1s budget → error "budget exceeded" with partial audit. vp test green.

M4 Scheduler — cancellation classes & teardown

Parallel execution; cancel vs report-only gates; total teardown; signal exits (130/143).
See: a failing stub-det (cancel class) + slow stub-agent → agent phase cancelled (gates failed: …), exit 1; SIGTERM → exit 143 + partial report on disk.

M5 Config loading & precedence

flags > project > user > built-in, deep-merged per setting.
See: a project failOn: warning flips a warning-only run 0→1; a malformed config exits 2 naming the line.

M6 Model routing — tiers & qualification check

Tier→model resolution against credentialed providers; preflight failure; unqualified-model warning.
See: fake "only provider X" resolves the robust tier to X; no provider → preflight fails with the actionable message, exit 2, before any phase runs.

M7 Specialists — composite phases

Parallel sub-agents roll up to one PhaseReport; per-specialist cost; one-fails-others-survive.
See: stub-composite (3 specialists, 1 forced to error) → one report with the 2 survivors' findings, each tagged, per-specialist cost.

M7.5 Cloudflare-validated Coordinator judge stage + risk classifier

A composite phase can run an optional coordinator (judge pass) over its specialist roll-up, and a deterministic risk classifier scales fan-out + coordinator activation. The most load-bearing noise filter at scale, and its cost dial. PRD #25/#26 · plan P8/P9
  • coordinator config + judge run through the existing AgentRunner seam — submission replaces the raw roll-up
  • provenance (survivors keep their specialist) + cost.coordinator
  • deterministic classify(diff, paths, config) → level (pure; fixture rule only — real thresholds are the code-review PRD's)
  • level → (specialist subset, coordinator on/off) wiring; inert with no rules declared
See: stub-composite with a coordinator + riskLevels → small diff runs 1 specialist / no judge; large diff runs all specialists + the judge, whose merged findings (one dedup, one drop) become the phase's findings, with cost.coordinator + the resolved level present. vp test green.

M8 Spec context & large-diff visibility

--prd/--task combine; semantic diff pre-filtering (strip lockfiles/minified/generated); partial-coverage warning when a diff exceeds budget.
See: --prd spec.md --task "X" → both in report.spec.sources; an oversized diff → partial-coverage listing excluded files.

M9 Human output & display polish

The human surface (loops already work via JSON since M1).
See: stet (no JSON) → grouped, colored, located findings + cost footer; --quiet suppresses passing phases; --show error filters display only.

  • sequential policy → config sugar after M4
  • Budget presets → multiplier over M3's default set
  • Churn-ranked subsetting → M8 ships file-order + the warning

Every deferral chosen because no other milestone depends on it.

05Reality-disagrees protocol

If implementation contradicts a PRD decision — a contract that can't be built as specified, a budget default wrong in practice, an SDK API differing from the POC, a guard needing a fourth case — stop and surface it: name the contradiction, bring it for a call (the PRD is amended + its decisions table updated in the same change), only then continue. The docs follow reality deliberately; a silent deviation makes every downstream doc lie.

Watch points: Pi SDK 0.78.x API vs the POC's usage · whether 120 turns suffices for stub-agent under budget · scope detection on detached HEAD / shallow clones.
