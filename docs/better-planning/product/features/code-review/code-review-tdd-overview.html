Area A — agreement-verify stage (the keystone, live-walked)

A net-new harness capability (PRD #12/#35) that every future phase inherits. Built on the harness-controlled-provenance pattern hardened in issue #48.

A·0 · boundary

Standalone verify.ts, called by composite.run() between roll-up and coordinator — not folded in.

→ independently testable; future phases inherit it.

A·1 · pipeline order

Verify-all → coordinator dedups (PRD-literal).

not chosen: dedup-first (merges before corroboration); deterministic pre-dedup deferred to eval.

A·2 · voter contract & tier

submit_verdict ternary; independence via distinct lenses (one runner ×N); robust tier v1.

decision 8 makes voter-tier a later dial → start at the high baseline.

A·3 · failure semantics

Partial: retry once → abstain (absolute threshold). Total: raw roll-up, confidence: low + verify-degraded warning, never gates.

a broken verify must never manufacture a gating signal.

A·4 · protected class

confidence: high (3/3) joins evidence-backed as undroppable; confidence harness-stamped by id (downgrade impossible); separate audit.verify.

dropping a corroborated finding = missed gate, the costliest failure.

Verify — zoom on one candidate

Independence comes from the lenses, not from different models. One runner, N calls, each a fresh conversation with a different refutation angle.

one candidate finding
fan out ×3 (independent)
voter 1 · reproduction / soundness
voter 2 · partial-context skepticism
voter 3 · scope / blocklist
▼ count upholds (refute & abstain = not-uphold)
3 → high · 2 → medium · ≤1 → dropped

Area B — Finding contract evolution

B·1 · confidence ownership walked

Operative confidence = harness-owned (from verify). Specialist self-rating kept as labeled meta.selfConfidence — never shown to voters, used for nothing in v1, recorded for the eval to study.

self-reported confidence is the miscalibrated thing voting replaces; but discard-vs-use is empirical → measure it.

B·2 · pre-existing detection

Deterministic diff added-line membership (PRD #14). Location in added + hunks ⇒ introduced; else meta.preexisting. No line ⇒ not pre-existing (still gates).

B·3 · schema split

New SpecialistSubmission (no confidence/specialist/phase) vs shared Finding; additions meta-namespaced (non-breaking).

Area C — eval-suite subsystem (milestone 1, highest unknowns)

C·1 · execution model walked

Tiered split. Default vp test → cassettes (fast, deterministic, CI-safe). Opt-in eval:live (keyed) → real models, re-measures quality, updates a committed baseline.

two jobs (machinery vs quality), each in its own mode. Open: tiny live smoke-set per commit?

C·2 · grader

LLM-judge → HIT / VALID / NOISE; location gate (±N) + embedding match, 1-to-1; validated vs human golden subset κ≥0.75.

C·3 · metrics & gate

SNR = (HIT+VALID)/NOISE + per-tier P/R + clean-FPR. Committed eval-baseline.json; live gate fails on drop beyond ε.

C·4 · fixtures

PRD C5 shape; hand-authored across visibility tiers + clean + a few mined; under src/eval/; full mining → eval-suite feature.
