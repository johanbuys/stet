## 2026-06-23 - T13: bugs specialist + review phase factory + activation

**What was done:** Created `src/phases/review/review.ts` with the `bugs` SpecialistConfig and `makeReviewPhase` factory. Added `severityCeiling` and `maxFindings` optional fields to `SpecialistConfig` interface in composite.ts (backward-compatible — no enforcement yet, communicated via rubric text per TDD D). Created `src/phases/review/review.test.ts` with 22 tests verifying all accept criteria.

**Key decisions:**
- `submitSchema = SpecialistSubmission` is declarative: the fake runners in unit tests return full `Finding` objects (with confidence) so `parseFindings` passes. The real wiring (parseSpecialistSubmissions + confidence stamping before verify) is deferred; the TDD note says "Not yet wired (M4)" — this happens when T14 or a later step wires real runners. The unit tests verify structure, not parsing format.
- `severityCeiling` and `maxFindings` added to `SpecialistConfig` interface as optional fields. Not enforced in composite roll-up yet (TDD D says these are "mostly data"; enforcement via rubric text + future M6 gating).
- Activation is `ctx.scope.files.length > 0` — after M8 pre-filtering, `scope.files` already excludes stripped files, so any non-empty array means reviewable code exists.
- The 3 verify lenses map exactly to the TDD A·2 angles: reproduction/soundness, partial-context skepticism, scope/blocklist.
- `MAX_FINDINGS = 5` constant is substituted into the rubric at build time (template literal), not a runtime placeholder.
- The rubric text in review.ts is the shared preamble + bugs-specific section from `code-review-rubric-draft.md`, adapted for `MAX_FINDINGS=5` inline.

**Learnings:**
- The `SpecialistConfig` interface in composite.ts did not previously have `severityCeiling` or `maxFindings`. The PRD C2 spec listed them. They are additive optional fields — zero impact on existing phases that don't set them.
- The `FakeAgentRunner` in tests ignores `submitSchema` — it returns whatever is scripted regardless. So the "submitSchema=SpecialistSubmission" property on a specialist is a real-model concern, not a unit-test concern.
- Verify voters in the fake tests reuse a single FakeAgentRunner that returns the same verdict for every call — this works because `FakeAgentRunner` returns its scripted response on every `run()` call.
- `vp check --fix` was needed to fix formatting in review.test.ts (the formatter reordered the import of `SpecialistSubmission` to combine it with the type import from the same module).
