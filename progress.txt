**Learnings:**
- `AbortSignal.any([sig1, sig2])` is available in Bun 1.3.14 and propagates the reason of the
  first signal that fires. This is the right tool for merging external + internal abort signals.
- When `controller.abort()` is called without a string reason, `signal.reason` is a `DOMException`
  (not a string). The check `typeof signal.reason === "string"` guards against this.
- T14's test that asserted `receivedSignals[0] === controller.signal` had to change: the scheduler
  now wraps `ctx.signal` in `AbortSignal.any`, so phases receive a combined signal (different
  object). Updated to check it's an AbortSignal instance and that it fires when `ctx.signal` fires.
- The key PRD rule: only `status === "completed"` with error-severity findings counts as a gate
  "failure". `status === "error"` (timeout, spawn failure) is always report-only.
- T15 builds on T14's signal seam exactly as planned: scheduler fires `gateController`, agent
  phases receive it via `combinedSignal`, detect `ctx.signal.aborted` in the error path, and
  return `cancelled` with the reason from `signal.reason`.
