## 2026-06-14 - TASK-034: T28: Coordinator rubric fixture + provenance/gating interaction

**What was done:** Implemented constrained authority (PRD #30) in the coordinator judge pass:
- `composite.ts`: After the coordinator success path builds `finalFindings`, a new loop checks
  each raw roll-up finding that carries `evidence?.command` (deterministic/evidence-backed). If
  the coordinator dropped it (not in output) or downgraded its severity, the harness reinstates
  the original finding unchanged and records it in `audit.coordinator.reinstated`. The `dropped`
  list is computed AFTER reinstatement, so reinstated findings correctly appear in `findings` and
  not in `dropped`. Used `as const` for `SEVERITY_RANK` to satisfy TypeScript's strict-undefined
  check on indexed record access.
- `stub-composite.ts`: Extended `makeStubComposite(runners, opts?)` with an optional
  `{ coordinator?: CoordinatorConfig }` second argument, wired through to `makeCompositePhase`.
- `coordinator.test.ts`: Added 10 new T28 tests in two new describe blocks:
  1. "constrained authority" — drop scenario, downgrade scenario, non-protected drop accepted,
     reinstated not in dropped, schema compliance.
  2. "gating interaction" — AI-judgment downgrade stops gating at failOn=error, still gates at
     failOn=warning; protected finding reinstated at original severity still gates at failOn=error.
  Also imported `deriveExit` from `exit-codes.ts` for gating interaction tests.

**Learnings:**
- Dependencies (`bun install`) weren't installed when the session started — run `bun install`
  first to get the `node_modules/.bin/vp` binary, then use `node_modules/.bin/vp test`.
- `SEVERITY_RANK` typed as `Record<string, number>` still triggers TS2532 "possibly undefined"
  on index access in strict mode; use `as const` to get literal-typed keys that TypeScript
  can see are exhaustive.
- Reinstated findings must be excluded from `dropped` — compute `survivorIds` (and therefore
  `dropped`) AFTER running the reinstatement loop, not before.
- `finalFindings` produced by `.map()` is a regular mutable array; safe to `.push()` / index-
  assign into it for reinstatement without rebuilding.
